i18n?? l10n??

  • Aren't Acronymns fun?

    Internationalization - i18n

    Localization - l10n

    Globalization - g11n?!

  • You keep saying that word...

    Internationalization is the task of making an application adaptable to various regions / languages without design or engineering changes.

    Localization is the process of adapting internationalized applications for a specific language

    i18n is something that you, the developer do, while l10n is something that a translator does

    Think of it as i18n is what loads a language pack, while l10n is the language pack itself

But really, what does that mean?

  • Text

    Content Checking (Tone)

    String Literal Transitions

    Pluralization

    Sorting

    RTL Content Display

    Accessibility

  • Numbers / Money

    Comma Separator / Decimal Separator

    Currency Conversion

    Metric vs. Imperial Measurements

  • Dates / Times

    Date Formats

    12 Hour Clock vs. 24 Hour Clock

    Timezones

    Start of Week

Cool, so how do I do all that?

  • Server Side i18n

    Compile a full application per locale/region required

    Changing locales requires a full page reload

    Extra server logic to serve appropriate version of application

    Serve only exactly what you need

    Write it yourself of integrate with existing service (like CMS)

  • Client Side i18n

    Application and language packs are loaded / cached on initial load

    Switch languages on the fly without page reload

    Additional logic / markup required in application code

    Lots of 3rd Party helper libraries out there (especially with Angular)

    Full disclosure: This is the approach we've taken at Message Systems
